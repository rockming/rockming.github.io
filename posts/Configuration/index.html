    Win10 Pro Install Tensorflow Gpu

    December 27, 2020

    Window10专业版配置机器学习环境 # 基础信息 # win10专业版 GTX1050Ti pip 20.3.1 conda 4.9.2 目标配置 # sklearn + tensorflow-gpu + keras 问题描述 # 由于之前在Ubuntu20.04上通过anaconda3成功安装了sklearn+tensorflow-gpu+keras，想在win10上采用相同的方法进行安装，意料之外的是出现了版本兼容问题，主要是sklearn与tensorflow-gpu之间的版本冲突。通过两天的研究发现，直接通过conda安装scikit-learn时，会安装最新的版本（0.23.0）与此同时会安装高版本的numpy（1.19.2），scipy（1.5.2），这就导致了在导入sklearn时出现问题。然后我想可以通过安装低版本的scikit-learn（如0.20.0），安装完成后发现确实可以直接导入sklearn了。（但是有问题！！！） 本以为通过降级的方式就可以解决问题了，但是当我直接通过conda安装tensorflow-gpu=2.0.0的时候，与其相关的高版本numpy依赖也会被重新安装并且会覆盖刚才降级后的版本，且安装后无法正常使用gpu。这时候再次在ipython中导入sklearn的时候发现又出现了上述问题。一直在这个地方困扰了很久，也是一度要放弃了。 通过多次的新建虚拟环境，重新安装，并与Ubuntu20.04中安装的组件版本对比之后发现了上述问题。以下为我的解决方案介绍，仅供参考。 解决方案 # 思路：混合使用conda和pip安装 (尽管这种方式不推荐，但是真正能够解决问题）。 # 通过conda创建虚拟环境，这里创建的Python版本为3.7，并进入该环境，（据说该版本对tensorflow-gpu支持性比较好） conda create -n keras python==3.7 conda activate Keras 首先安装低版本的numpy（1.18.5），scipy（1.4.1），scikit-learn （0.23.2） pip install numpy==1.18.5 scipy==1.4.1 scikit-learn==0.23.2 安装cuda，以及神经网络的计算库cudnn。这个可以直接通过conda或者pip安装。这个可以看一下自己显卡驱动版本以及对应cudatoolkit，cudnn。如果直接安装tensorflow-gpu=2.0.0 则默认安装cudatoolkit=10.0.130和cudnn=7.6.5。这个好像在win10上有问题，会缺乏dynamic library cudart 64_101.dll，这就导致gpu无法正常使用。 conda install cudatoolkit==10.1.243 cudnn==7. ...
